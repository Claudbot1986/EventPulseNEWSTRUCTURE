```

       ↓

       ↓

       ↓
C4-post-hoc: VARFÖR gapet?
       ↓
Skapa generell regel
       ↓
━━━ ITERATIV FÖRBÄTTRINGSLOOP (max 3 iterationer) ━━━━━━━━━━━
┃
┃  ITERATION 1:
┃  → Gate väljer candidate
┃  → Verification Batch
┃  → Regression Batch
┃  → Gate: keep / refine / rollback
┃
┃  ITERATION 2 (om refine):
┃  → Förfina regeln
┃  → Verification + Regression
┃  → Gate: keep / refine / rollback
┃
┃  ITERATION 3 (final, om refine):
┃  → Sista försöket
┃  → Gate: keep / rollback
┃
┃  STOPP:
┃  → 3 iterationer utan keep → manual-review
┃  → 3 iterationer med keep → 强烈推荐 implementering
```

## 123-IMPROVEMENT-GATE

### Gate-regler

| # | Regel |
|---|-------|
| 1 | **Max 1 aktiv candidate** per source |
| 2 | **Max 3 iterationer** per source |
| 3 | **Matcha failure category** |
| 4 | **Blocker-kontroll** |
| 5 | **Gate-outputs:** selected / blocked / rejected / keep / rollback / refine |

### Blocker-typer

| Blocker | Beskrivning |
|---------|-------------|
| `source_exhaustion` | Source lämnade poolen |
| `attribution_loss` | Effekten kan inte tillskrivas |
| `regression_risk` | Försämrar existerande routing |
| `scope_mismatch` | Site-specifik, inte generaliserbar |

### Beslut-logik

| Beslut | Villkor |
|--------|---------|
| **SELECTED** | Passerar alla gate-regler |
| **BLOCKED** | Source har redan aktiv improvement |
| **REJECTED** | Site-specifik / låg confidence / låg impact |
| **KEEP** | Verification + Regression passed |
| **REFINE** | Partial success, kan raffineras |
| **ROLLBACK** | Failed verification/regression |

## ITERATIV FÖRBÄTTRINGSLOOP

### Loop-flow (upp till 3 iterationer)

```
FÖR VARJE source med gap:
  ┌─────────────────────────────────────┐
  │ ITERATION 1                         │
  │ → Gate väljer candidate              │
  │ → Applicera + Verification Batch     │
  │ → Regression Batch                   │
  │ → Gate: keep / refine / rollback    │
  └─────────────────────────────────────┘
             ↓ (refine)
  ┌─────────────────────────────────────┐
  │ ITERATION 2                         │
  │ → Förfina regeln                    │
  │ → Verification + Regression          │
  │ → Gate: keep / refine / rollback    │
  └─────────────────────────────────────┘
             ↓ (refine)
  ┌─────────────────────────────────────┐
  │ ITERATION 3 (final)                 │
  │ → Maximalt förfinad regel            │
  │ → Verification + Regression          │
  │ → Gate: keep / rollback             │
  └─────────────────────────────────────┘
```

### Exit-criteria

| Villkor | Åtgärd |
|---------|--------|
| Gate `keep` | Regeln blir active,强烈的 implementeringsrekommendation |
| Gate `rollback` | Förkasta, source till manual-review |
| 3 iterationer utan keep | Source till manual-review, logga "exhausted" |
| Alla iterationer keep | → 强烈推荐 implementering |

### Verification Batch

- Testar på sources med **samma failure category**
- Minst 2 sources
- Krav för pass:
  - Mätbar effekt på ≥ 2 sources
  - Ingen regression på kontroll-sources
  - Effekten är attribuerbar

### Regression Batch

- Testar på **nya liknande sources**
- Minst 2 new sources
- Krav för pass:
  - Samma förbättring på new sources
  - Inget nytt gap skapas

## PROCESSÖVERSIKT (STEG 0-9)

```
STEG 0: Ingestion-Routing (alltid först)
  → Sortera källor: A/B/C/D/E

STEG 1: Skapa initial testpool (max 10 från postB-preC)
  → poolRoundNumber = 1

STEG 2: C4-AI Ground Truth (före C3)
  → AI crawlar för ground truth
  → Output: "Denna sida har X events"

STEG 3: Kör C1→C2→C3 (rule-based)
  → C3 vs C4 → GAP identifieras

STEG 4: Exit-kontroll
  → events → postTestC-UI
  → A/B/D-signal → postTestC-A/B/D
  → 3 rundor utan events → manual-review

STEG 5: C4-Post-Hoc Analys
  → Analysera fail-mängden
  → Föreslå förbättringar (alla typer)

STEG 6: ITERATIV FÖRBÄTTRINGSLOOP (1-3 iterationer)
  → Gate väljer candidate
  → Verification Batch
  → Regression Batch
  → Gate: keep / refine / rollback
  → Vid refine: nästa iteration
  → Vid keep: 强烈推荐 implementering
  → Vid rollback: manual-review

STEG 7: Refill (mellan rundor)
  → Töm postB-preC helt
  → Om tom: stoppa, `no-sources-available`

STEG 8: Nästa runda
  → poolRoundNumber++
  → Upprepa Steg 2-7

STEG 9: Avslut
  → Skriv slutrapport
  → Uppdatera batch-state.jsonl: status="completed"
```
