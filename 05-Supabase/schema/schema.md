# 05-Supabase/schema

## Purpose

Documents the Supabase/PostgreSQL database schema for EventPulse: tables, columns, types, relationships, indexes, and constraints.

## Core Tables

### events

Primary event storage. Written by `04-Normalizer` via upsert logic.

Key columns:
- `id` (UUID, PK) ‚Äî generated by Supabase
- `title_en` / `title_sv` ‚Äî language-split titles
- `description_en` / `description_sv` ‚Äî language-split descriptions
- `start_time` / `end_time` ‚Äî ISO timestamps
- `source` ‚Äî event source (e.g. 'eventbrite', 'ticketmaster')
- `source_id` ‚Äî source-specific event ID
- `venue_id` (UUID, FK ‚Üí venues) ‚Äî resolved venue
- `lat` / `lng` ‚Äî coordinates (fallback to Stockholm center)
- `location` ‚Äî PostGIS POINT(lng lat)
- `is_free` ‚Äî boolean
- `price_min_sek` / `price_max_sek` ‚Äî price range
- `ticket_url` ‚Äî event or ticket URL
- `image_url` ‚Äî event image
- `category_slug` ‚Äî denormalized category for fast filtering
- `dedup_hash` ‚Äî SHA-256 for deduplication
- `status` ‚Äî always 'published' for normalized events
- `raw_data` ‚Äî original raw payload for debugging
- `updated_at` ‚Äî auto-updated on change

Indexes: likely on `source`, `start_time`, `venue_id`, `dedup_hash`, `category_slug`.

### venues

Venue records resolved by `04-Normalizer`.

Key columns:
- `id` (UUID, PK)
- `name` ‚Äî venue name
- `address` ‚Äî street address
- `lat` / `lng` ‚Äî coordinates
- `location` ‚Äî PostGIS POINT(lng lat)

### categories

Category lookup table. Stores slugs and UUIDs.

- `id` (UUID, PK)
- `slug` ‚Äî URL-safe category identifier (e.g. 'music', 'theater', 'community')

### event_categories

Many-to-many join between events and categories.

- `event_id` (UUID, FK ‚Üí events)
- `category_id` (UUID, FK ‚Üí categories)

### ingestion_logs

Per-source ingestion audit log.

- `id` (UUID, PK)
- `source` ‚Äî source name
- `action` ‚Äî 'inserted' or 'updated'
- `timestamp` ‚Äî ISO timestamp
